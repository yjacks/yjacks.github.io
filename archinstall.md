# Arch 基本使用教程

## 前言

务必确认你有折腾 Arch 的欲望，否则半途而退很正常。

本文以 FDL1.3 and newer 协议发布。

本教程假定你使用一台拥有显卡及显示屏，并且你在前方操作的电脑。远程安装不在本教程范围内。

## 前期准备

## 部分术语

基本系统：可以运行的系统，仅携带基本软件。

软件包：被打包的软件。使用 Pacman 软件包管理器的系统下，这种文件拓展名多为 `.pkg.tar.xx` (现在通行的为 `.pkg.tar.zst`，也有少部分使用 `.pacman/.pkg.tar.xz` 拓展名）。

包管：全称软件包管理器，系统中用于管理软件包的软件。是各个发行版的主要差别之一。Arch 使用 Pacman 作为包管。

发行版：在此处指 Linux 发行版。是各个使用
Linux 作为内核的系统分支。

### 基本系统运行所需硬件

- amd64 架构 CPU
- 足够运行一个基本系统的内存（600MiB 左右）
- 支持“文本模式”输出的显卡（八十年代后的显卡基本都支持）(缺失显卡的称为“无头系统”）
- 至少 2GiB 的硬盘空间，用来存放基本系统
- 最基本的外设（键盘、 显示屏）

## 安装开始

### 下载、校验及刻录安装映像

* 本教程假定你使用另一台电脑，或使用将要安装系统的电脑上原有的系统完成本步骤。*

#### 下载
 
本教程推荐你可能的情况下使用 BT 下载。考虑到有时 BT 下载在一些情况下不可用，也会介绍 HTTP 下载。

##### BT 下载

访问 [官网下载页](https://archlinux.org/download/)，找到 "BitTorrent Download (recommended)" 字样。

下面有两个超链接，选择“Torrent for xxxx.xx.xx”（日期，格式为年 . 月 . 日，意思是安装映像的发布时间）并下载。

打开任何一个 BT 下载软件（如 BitComet、Transmission、QBitTorrent 等）（Aria2/Motrix、FDM稍慢）（尽量不要使用迅雷，因为大陆纯 BT 网络可以支持该镜像满速下载），用他们打开下载下来的文件，等待下载完成。

##### HTTP 下载

访问 [官网下载页](https://archlinux.org/download/)，找到 "HTTP Direct Downloads" 下的 "China"（或者你所在的国家）（如果你所在的国家没有镜像，请使用临近国家的镜像。如，如果你在朝鲜，请使用韩国、中国或俄罗斯的镜像）。

在下面的超链接（蓝字）中随意选择一个点入，你应该会看到一个树状结构，请选择正数第二个文件（archlinux-xxxx.xx.xx-x86_64.iso）来下载它。

#### 校验

首先，校验对我们的意义是：确定文件网络传输时，是否被篡改或未完全传输。

要校验一个镜像，我们需要知道它在原本情况下镜像的摘要值。我们选择 SHA256（一种摘要算法） 算出的摘要值校验。

在下载页（同前）找到 "HTTP Direct Download"，然后找到 "Checksums" 下的 "SHA256" 栏，它的值像是这样：

> 61dbae312cf677be38a93f424c91abadd8a8ed1f3a602b697aac4c57a7872645

（2023.01.01 镜像的 SHA256 摘要值）

接下来需要分系统操作。

##### Windows

同时按下 "Windows" 键与 "R" 键，在弹出的界面中输入 "cmd" 并回车。在弹出的黑色窗口（Windows 下的终端模拟器，这个概念之后专门解释）中，输入

``` certutil 你存放镜像的位置 SHA256```

稍等，"cmd" 将会输出一段字符串，这就是下载下来的镜像的 SHA256 摘要值。将它与在下载页获得的 SHA256 对比，如果一样就是正确的，可以进入下一步。

##### GNU/Linux

打开你使用系统的终端模拟器，输入

``` sha256sum 你存放镜像的位置 ```

对比输出与下载页提供的摘要值，一样就可进入下一步。

##### BSD
 
打开终端模拟器，输入

``` cksum -a sha256 镜像位置 ```

对比输出与下载页提供的摘要值，一样就可进入下一步。
